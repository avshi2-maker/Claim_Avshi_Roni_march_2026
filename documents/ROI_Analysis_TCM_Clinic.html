סיכום מנהלים

פרויקט: אפליקציה לניהול קליניקת דיקור עם בינה מלאכותית

מודל עסקי: מנוי חודשי למדקרים ($19-29 למשתמש)

שותפות: 20% אבשי / 80% ד"ר רוני ספיר

תקופת תחזית: 5 שנים

1. ניתוח עלויות למדקר בודד (הוצאות משתנות)

כמות שאילתות למדקר: 5 שאילתות × 10 לקוחות × 22 ימי עבודה = 1,100 שאילתות בחודש

⚠️ נורת אזהרה: במחיר מנוי של $8, העלות הישירה ($11 API + $1.2 עמלה = $12.2) גבוהה מההכנסה. במצב זה, ככל שיהיו יותר לקוחות, כך נפסיד יותר כסף.

פתרון: עדכון מחיר מנוי ל-$19-29

2. מבנה הוצאות קבועות חודשי (OPEX)

סעיף עלות חודשית הערות
תחזוקה וגיבויים $50 עבודת פיתוח שוטפת
אחסון ושרתים (Cloud) $30 בסיס נתונים ואחסון

3. תוכנית ROI רב-שנתית (מחיר מנוי $19)

א. תקופת הרצה (חודשים 1-3)

  • יעד: 20 משתמשים ראשונים (Beta)
  • הכנסות: $380 לחודש
  • הוצאות: $588 (קבועות) + $244 (משתנות)
  • שורה תחתונה: הפסד של ~$450 בחודש (שלב למידה)

ב. שנה ראשונה (צמיחה)

  • יעד: 200 מדקרים (פריסה עולמית)
  • הכנסות שנתיות: $45,600
  • הוצאות (כולל שיווק): $35,000
  • ROI: הגעה לנקודת איזון (Break-even) סביב חודש 8
  • חלק אבשי (20%): ~$2,120 רווח בשנה ראשונה

ג. שנה שלישית (Scale)

  • יעד: 1,000 משתמשים
  • הכנסה נקייה: ~$50,000 בשנה (אחרי כל ההוצאות)
  • חלק אבשי (20%): $10,000

ד. שנה חמישית (בגרות)

  • יעד: 5,000 משתמשים
  • סטטוס: הפיכה לסטנדרט בתחום
  • רווח תפעולי: גבוה בזכות אופטימיזציה של API
  • הערכה משמרת (חלק אבשי 20%): $15,000-20,000 לשנה

4. המלצות אסטרטגיות

1. מודל תמחור

מחיר של $8 נמוך מדי עבור כלי מבוסס API אינטנסיבי. מומלץ $19-29 לחודש, או הגבלת מספר שאילתות במחיר הבסיס.

2. ייעול טכני

שימוש ב-Caching (שמירת תשובות לשאלות נפוצות) לחיסכון בקריאות API.

3. פילוח שוק

התמקדות במדקרים דוברי אנגלית (ארה"ב/אירופה) - נכונות גבוהה יותר לתשלום.
